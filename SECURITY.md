# Security Policy

## Сообщение Об Уязвимости

Пожалуйста, не публикуйте рабочий эксплойт или чувствительные сведения в открытом issue. Для сообщения об уязвимости используйте private vulnerability reporting в GitHub, если он включен для репозитория, либо свяжитесь с владельцем репозитория через профиль GitHub.

В сообщении полезно указать версию или commit, шаги воспроизведения, ожидаемое влияние и безопасный пример данных.

## Границы Проекта

Этот проект является эвристическим read-only инструментом для первичного анализа. Он не гарантирует обнаружение вредоносного ПО и не должен использоваться как замена антивирусу, EDR или ручному расследованию инцидента.
