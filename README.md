# Windows Suspicious File Audit

Read-only инструмент для первичного поиска подозрительных признаков в Windows. Он проверяет распространенные механизмы закрепления, процессы и рискованные файлы в профиле пользователя, формирует отчеты и при наличии Kaspersky может передать отдельные находки на локальную дополнительную проверку.

Это не полноценный антивирус и не средство автоматического лечения. Скрипт не удаляет файлы, не помещает их в карантин и не отправляет хэши во внешние сервисы.

## Возможности

- проверка `Run`/`RunOnce`, папок Startup, сторонних scheduled tasks, auto-start services и WMI consumers;
- выявление процессов и команд, запускаемых из `Temp`, `AppData`, `LocalAppData`, `Downloads` и `Desktop`;
- анализ исполняемых файлов и скриптов по подписи, SHA-256, скрытому атрибуту, возрасту и обманным двойным расширениям;
- извлечение payload из команд `powershell`, `wscript`, `cscript`, `mshta` и `rundll32`;
- HTML, JSON и CSV отчеты с историей новых и исчезнувших находок;
- безопасная история: неполный аудит не заменяет baseline, а разные области сканирования хранятся отдельно;
- опциональная локальная перепроверка находок через Kaspersky CLI.

## Требования

- Windows PowerShell 5.1 или PowerShell 7;
- Windows 10/11 для системных источников данных;
- Kaspersky с доступным `avp.com` только для режима дополнительной проверки;
- Pester для запуска тестов.

## Быстрый Старт

Запустите один из готовых сценариев:

```bat
Start-SuspiciousFileAudit.cmd
Start-DeepAudit.cmd
```

Или вызовите основной скрипт непосредственно:

```powershell
.\Invoke-SuspiciousFileAudit.ps1 -VerifyFindingsWithKaspersky
.\Invoke-SuspiciousFileAudit.ps1 -AuditProfile Deep -IncludeLargeUserFolders -MaxFiles 100000
```

Результаты сохраняются локально:

- `reports\report-*.html` - удобный отчет для просмотра;
- `reports\report-*.json` - полная машинно-читаемая сводка;
- `reports\findings-*.csv` - таблица находок;
- `state\last-audit-*.json` - baseline для сравнения следующих полных запусков.

Папки `reports`, `state` и `security-audit-output` исключены из Git, поскольку могут содержать сведения о локальной системе и путях пользователя.

## Режимы

### Быстрая Проверка

Проверяет системные источники закрепления, запущенные процессы и рискованные типы файлов в `Downloads`, `Desktop` и временных каталогах.

```powershell
.\Invoke-SuspiciousFileAudit.ps1
```

### Глубокая Проверка

Добавляет обход `AppData` и `LocalAppData`. Такой запуск может быть значительно дольше и создавать больше находок для ручной проверки.

```powershell
.\Invoke-SuspiciousFileAudit.ps1 -AuditProfile Deep -IncludeLargeUserFolders -MaxFiles 100000
```

### Недельная Проверка

Создать scheduled task для воскресенья в 12:00:

```powershell
.\Install-WeeklyAudit.ps1
```

Создать глубокую еженедельную проверку:

```powershell
.\Install-WeeklyAudit.ps1 -Deep
```

Удалить задачу:

```powershell
.\Remove-WeeklyAudit.ps1
```

## Интерпретация Результатов

- `High` - сильный эвристический индикатор, например двойное исполняемое расширение или закодированная/удаленная команда.
- `Medium` - объект требует проверки: неподписанный файл, пользовательский путь запуска или постоянный WMI consumer.
- `Low` и `Info` - контекст для расследования, а не утверждение о вредоносности.

`AuditComplete: false` означает, что сканирование было ограничено, например достигнут `MaxFiles` или часть источников недоступна. В этом случае история намеренно не обновляется и исчезнувшие находки не вычисляются.

## Тесты

```powershell
Import-Module Pester
Invoke-Pester .\Tests\Invoke-SuspiciousFileAudit.Tests.ps1
```

Тесты покрывают дедупликацию находок по пути, двойные расширения, безопасное обновление baseline, разделение областей аудита и разбор launcher-команд.

## Безопасность И Приватность

Инструмент работает локально и предназначен для triage, а не для автоматических решений. Перед публикацией отчета просмотрите его содержимое: JSON/CSV/HTML могут раскрывать имя компьютера, пользователя, пути и хэши файлов.

Об уязвимостях самого проекта сообщайте согласно [SECURITY.md](SECURITY.md).
